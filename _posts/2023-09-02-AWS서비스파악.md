### 분석

- Amazon Athena
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Athena는 표준 SQL을 사용하여 Amazon S3의 데이터를 쉽게 분석할 수 있게 해주는 대화형 쿼리 서비스입니다. Athena는 서버리스이므로 설정하거나 관리할 인프라가 없으며 실행한 쿼리에 대해서만 비용을 지불합니다. 시작하려면 S3의 데이터를 가리키고 스키마를 정의한 다음 표준 SQL을 사용하여 쿼리를 시작하면 됩니다.
</pre>

- AWS Data Exchange
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
AWS Data Exchange는 고객이 AWS 클라우드에서 타사 데이터를 쉽게 찾고, 구독하고, 사용할 수 있게 해주는 서비스입니다.
</pre>
- AWS Data Pipeline
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
AWS Data Pipeline은 데이터 이동 및 변환을 자동화하는 데 사용할 수 있는 웹 서비스입니다. AWS Data Pipeline을 사용하면 데이터 기반 워크플로를 정의할 수 있으므로 이전 작업의 성공적인 완료에 따라 작업이 달라질 수 있습니다.
</pre>
- Amazon EMR
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon EMR은 Apache Hadoop과 Amazon Web Services에서 제공하는 서비스를 사용하여 방대한 양의 데이터를 쉽고 효율적으로 처리할 수 있게 해주는 웹 서비스입니다.
</pre>
- AWS Glue
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS Glue는 분석, 기계 학습 및 애플리케이션 개발을 위해 데이터를 쉽게 검색, 준비 및 결합할 수 있게 해주는 확장 가능한 서버리스 데이터 통합 ​​서비스입니다.
</pre>
- Amazon Kinesis
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon Kinesis를 사용하면 비디오 및 데이터 스트림을 실시간으로 쉽게 수집, 처리 및 분석할 수 있습니다.
</pre>
- AWS Lake Formation
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS Lake Formation은 데이터 레이크를 쉽게 설정, 보호 및 관리할 수 있게 해주는 관리형 서비스입니다. Lake Formation은 데이터 원본을 검색한 다음 데이터를 분류, 정리 및 변환하는 데 도움이 됩니다. Lake Formation을 사용하여 데이터를 보호하고 Amazon Simple Storage Service(Amazon S3) 데이터 레이크로 수집할 수 있습니다.
</pre>
- Amazon Managed Streaming for Apache kafka(Amazon MSK)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon Managed Streaming for Apache Kafka(Amazon MSK)는 Apache Kafka를 사용하여 스트리밍 데이터를 처리하는 애플리케이션을 쉽게 구축하고 실행할 수 있게 해주는 완전관리형 서비스입니다.
</pre>
- Amazon OpenSearch Service(Amazon Elasticsearch Service)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon OpenSearch Service는 인기 있는 오픈 소스 검색 및 분석 엔진인 OpenSearch를 쉽게 배포, 운영 및 확장할 수 있게 해주는 관리형 서비스입니다. OpenSearch 서비스는 또한 보안 옵션, 고가용성, 데이터 내구성 및 OpenSearch API에 대한 직접 액세스를 제공합니다.
</pre>
- Amazon QuickSight
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon QuickSight는 시각화를 구축하고, 임시 분석을 수행하고, 데이터에서 신속하게 비즈니스 통찰력을 얻을 수 있는 빠른 비즈니스 분석 서비스입니다. Amazon QuickSight는 AWS 데이터 소스를 원활하게 검색하고, 조직이 수십만 명의 사용자로 확장할 수 있도록 지원하며, Amazon QuickSight SPICE(초고속, 병렬, 인 메모리, 계산 엔진)를 사용하여 빠르고 응답성이 뛰어난 쿼리 성능을 제공합니다.
</pre>
- Amazon Redshift
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon Redshift는 기존 비즈니스 인텔리전스 도구를 사용하여 모든 데이터를 간단하고 비용 효율적으로 효율적으로 분석할 수 있게 해주는 빠르고 완벽하게 관리되는 페타바이트 규모의 데이터 웨어하우스 서비스입니다. 이는 수백 기가바이트에서 페타바이트 이상의 데이터 세트에 최적화되어 있으며 비용은 연간 테라바이트당 $1,000 미만입니다. 이는 대부분의 기존 데이터 웨어하우징 솔루션 비용의 10분의 1입니다.
</pre>

### 애플리케이션 통합
- Amazon AppFlow
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon AppFlow는 SaaS(Software as a Service) 애플리케이션을 AWS 서비스에 연결하고 데이터를 안전하게 전송하는 데 사용하는 완전관리형 API 통합 서비스입니다. Amazon AppFlow 흐름을 사용하면 코드를 작성할 필요 없이 데이터 전송을 관리하고 자동화할 수 있습니다.
</pre>
- AWS AppSync
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS AppSync는 실시간 데이터 동기화 및 오프라인 프로그래밍 기능을 갖춘 엔터프라이즈 수준의 완전 관리형 GraphQL 서비스입니다.
</pre>
- Amazon EventBridge(Amazon CloudWatch Events)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon EventBridge는 애플리케이션을 다양한 소스의 데이터와 쉽게 연결할 수 있게 해주는 서버리스 이벤트 버스 서비스입니다. EventBridge는 자체 애플리케이션, SaaS(Software-as-a-Service) 애플리케이션 및 AWS 서비스에서 실시간 데이터 스트림을 제공하고 해당 데이터를 AWS Lambda와 같은 대상으로 라우팅합니다. 모든 데이터 소스에 실시간으로 반응하는 애플리케이션 아키텍처를 구축하기 위해 데이터를 보낼 위치를 결정하는 라우팅 규칙을 설정할 수 있습니다. EventBridge를 사용하면 느슨하게 결합되고 분산된 이벤트 기반 아키텍처를 구축할 수 있습니다.
</pre>
- Amazon MQ
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon MQ는 클라우드에서 메시지 브로커를 쉽게 설정하고 운영할 수 있게 해주는 관리형 메시지 브로커 서비스입니다. Amazon MQ는 기존 애플리케이션 및 서비스와의 상호 운용성을 제공합니다. Amazon MQ는 자체 메시징 시스템을 관리, 운영 또는 유지할 필요 없이 기존 애플리케이션 및 서비스와 함께 작동합니다.
</pre>
- Amazon Simple Notification Service(Amazon SNS)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon Simple 알림 서비스(Amazon SNS)는 애플리케이션, 최종 사용자 및 디바이스가 클라우드에서 즉시 알림을 보내고 받을 수 있도록 하는 웹 서비스입니다.
</pre>
- Amazon Simple Queue Service(Amazon SQS)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon Simple Queue Service(Amazon SQS)는 마이크로서비스, 분산 시스템 및 서버리스 애플리케이션을 쉽게 분리하고 확장할 수 있게 해주는 완전 관리형 메시지 대기열 서비스입니다. Amazon SQS는 분산 애플리케이션 구성 요소 간에 데이터를 이동하고 이러한 구성 요소를 분리하는 데 도움이 됩니다.
</pre>
- AWS Step Function
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS Step Functions를 사용하면 분산 애플리케이션의 구성 요소를 시각적 워크플로의 일련의 단계로 쉽게 조정할 수 있습니다. 상태 시스템을 신속하게 구축하고 실행하여 안정적이고 확장 가능한 방식으로 애플리케이션의 단계를 실행할 수 있습니다.
</pre>

### AWS 비용 관리
- AWS Budgets
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- 유동적인 예산 및 예측 기능을 사용해 계획 및 비용 제어 과정 개선
- AWS Budgets를 사용하면 사용자 지정 예산을 설정하여 비용 및 사용량을 추적하고, 임계값 초과 시 이메일 또는 SNS 알림에서 수신된 알림에 빠르게 대응할 수 있습니다.
</pre>
- AWS Cost and Usage Report
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS 비용 및 사용량 데이터에 대해 자세히 알아보기
- AWS Cost & Usage Report(CUR)를 사용하면 계정에 대한 가장 포괄적인 비용 및 사용량 데이터를 검토하고 항목화하며 정리할 수 있습니다.
</pre>
- AWS Cost Explorer
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- 시간에 따른 AWS 비용 및 사용량의 시각화, 이해 및 관리
- AWS Cost Explorer는 시간에 따른 AWS 비용과 사용량을 시각화, 이해 및 관리할 수 있는 손쉬운 인터페이스를 제공합니다. 비용 및 사용량 데이터를 분석하는 사용자 지정 보고서를 작성하여 신속하게 시작합니다. 데이터를 높은 수준으로 분석(예: 모든 계정의 총 비용 및 사용량)하거나 비용 및 사용량 데이터를 자세히 분석하여 추세를 식별하고 비용 동인을 파악하고 이상을 탐지합니다.
</pre>
- Saving Plans
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Savings Plans는 Amazon EC2 및 Fargate 사용량을 상당한 비율로 절약하는 데 도움이 되는 유연한 가격 모델입니다. Savings Plans는 1년 또는 3년 동안 일관된 사용량을 약속하는 대가로 Amazon EC2 및 Fargate에 저렴한 가격을 제공합니다.
</pre>

### 컴퓨팅
- AWS Batch
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS Batch를 사용하면 AWS 클라우드에서 배치 컴퓨팅 워크로드를 실행할 수 있습니다. 배치 컴퓨팅은 개발자, 과학자, 엔지니어가 대량의 컴퓨팅 리소스에 액세스하는 일반적인 방법입니다. AWS Batch는 필요한 인프라를 구성하고 관리하는 획일적인 부담을 제거합니다.
</pre>
- Amazon EC2
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon Elastic Compute Cloud(Amazon EC2)는 소프트웨어 시스템을 구축하고 호스팅하는 데 사용되는 크기 조정 가능한 컴퓨팅 용량, 즉 Amazon 데이터 센터의 서버를 제공하는 웹 서비스입니다.
</pre>
- Amazon EC2 Auto Scaling
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS는 애플리케이션을 확장하는 데 사용할 수 있는 여러 서비스를 제공합니다. Auto Scaling은 Amazon CloudWatch에 의해 활성화되며 CloudWatch 및 사용하는 기타 AWS 리소스에 대한 서비스 요금 외에 추가 비용 없이 사용할 수 있습니다.
</pre>
- AWS Elastic Beanstalk
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS Elastic Beanstalk를 사용하면 해당 애플리케이션을 실행하는 인프라에 대해 걱정할 필요 없이 AWS 클라우드에서 애플리케이션을 신속하게 배포하고 관리할 수 있습니다. AWS Elastic Beanstalk는 선택이나 제어를 제한하지 않고 관리 복잡성을 줄여줍니다. 애플리케이션을 업로드하기만 하면 AWS Elastic Beanstalk가 용량 프로비저닝, 로드 밸런싱, 조정 및 애플리케이션 상태 모니터링에 대한 세부 정보를 자동으로 처리합니다.
</pre>
- AWS Outposts
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS Outposts는 기본 AWS 서비스, 인프라 및 운영 모델을 거의 모든 데이터 센터, 코로케이션 공간 또는 온프레미스 시설에 제공합니다. 동일한 서비스, 도구 및 파트너 솔루션을 사용하여 클라우드 및 온프레미스용으로 개발할 수 있습니다.
</pre>
- AWS Serverless Application Repository
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS Serverless Application Repository는 서버리스 애플리케이션을 위한 관리형 리포지토리입니다. 이를 통해 팀, 조직 및 개인 개발자는 서버리스 아키텍처를 검색, 배포, 게시, 공유, 저장하고 쉽게 조립할 수 있습니다.
</pre>
- VMWare Cloud on AWS
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- 모든 VMware 워크로드를 위한 가장 빠르고 안전한 길
- 소프트웨어 제작사인 VMware와 선도적인 퍼블릭 클라우드 제공업체인 AWS가 완벽하게 지원하고 바로 실행 가능한 서비스로 컴퓨팅, 네트워크 및 스토리지 기능을 결합한 관리형 서비스를 통해 비즈니스 전환 목표를 빠르게 달성하세요.
</pre>
- AWS Wavelength
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- AWS Wavelength를 사용하면 개발자는 모바일 장치와 최종 사용자에게 매우 짧은 지연 시간을 제공하는 애플리케이션을 구축할 수 있습니다. Wavelength는 통신 사업자의 5G 네트워크 엣지에 표준 AWS 컴퓨팅 및 스토리지 서비스를 배포합니다. 개발자는 Amazon Virtual Private Cloud(VPC)를 하나 이상의 Wavelength Zone으로 확장한 다음 Amazon Elastic Compute Cloud(EC2) 인스턴스와 같은 AWS 리소스를 사용하여 매우 짧은 지연 시간과 해당 지역의 AWS 서비스에 대한 연결이 필요한 애플리케이션을 실행할 수 있습니다. .
</pre>

### 컨테이너
- Amazon Elastic Container Registry(Amazon ECR)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Elastic Container Registry(Amazon ECR)는 개발자가 Docker 컨테이너 이미지를 쉽게 저장, 관리 및 배포할 수 있게 해주는 완전관리형 Docker 컨테이너 레지스트리입니다.
</pre>
- Amazon Elastic Container Service(Amazon ECS)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">

### 데이터베이스
- Amazon Aurora
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Relational Database Service(Amazon RDS)는 클라우드에서 관계형 데이터베이스를 더 쉽게 설정, 운영 및 확장할 수 있게 해주는 웹 서비스입니다. 업계 표준 관계형 데이터베이스에 비용 효율적이고 크기 조정이 가능한 용량을 제공하고 일반적인 데이터베이스 관리 작업을 관리합니다. Amazon Aurora는 클라우드용으로 구축되었으며 MySQL 및 PostgreSQL과 호환되는 완전관리형 관계형 데이터베이스 엔진입니다. Amazon Aurora는 Amazon RDS의 일부입니다.
</pre>
- Amazon Aurora Serverless
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Aurora Serverless는 Amazon Aurora의 온디맨드 자동 크기 조정 구성 버전입니다. 애플리케이션 요구 사항을 기반으로 자동으로 시작 및 종료되어 용량을 확장 또는 축소합니다. 데이터베이스 인스턴스를 관리하지 않고도 클라우드에서 데이터베이스를 실행할 수 있습니다. 또한 기존 또는 새 데이터베이스 클러스터에서 프로비저닝된 인스턴스와 함께 Aurora Serverless v2 인스턴스를 사용할 수 있습니다.
</pre>
- Amazon DocumentDB (with MongoDB compatibility)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon DocumentDB(MongoDB 호환)는 빠르고 안정적이며 완벽하게 관리되는 데이터베이스 서비스로, MongoDB 호환 데이터베이스를 쉽게 설정, 운영 및 확장할 수 있습니다.
</pre>
- Amazon DynamoDB
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon DynamoDB는 원활한 확장성과 함께 빠르고 예측 가능한 성능을 제공하는 완전 관리형 NoSQL 데이터베이스 서비스입니다. Amazon DynamoDB를 사용하면 원하는 양의 데이터를 저장 및 검색하고 모든 수준의 요청 트래픽을 처리할 수 있는 데이터베이스 테이블을 생성할 수 있습니다. Amazon DynamoDB는 테이블의 데이터와 트래픽을 충분한 수의 서버에 자동으로 분산시켜 고객이 지정한 요청 용량과 저장된 데이터 양을 처리하는 동시에 일관되고 빠른 성능을 유지합니다.
</pre>
- Amazon ElastiCache
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon ElastiCache를 사용하면 AWS 클라우드에서 분산된 인 메모리 캐시 환경을 쉽게 설정, 관리 및 확장할 수 있습니다. 고성능, 크기 조정 가능하고 비용 효과적인 인메모리 캐시를 제공하는 동시에 분산 캐시 환경 배포 및 관리와 관련된 복잡성을 제거합니다. ElastiCache는 Redis 및 Memcached 엔진 모두에서 작동합니다. 어느 것이 가장 적합한지 확인하려면 사용자 가이드의 Memcached와 Redis 비교 항목을 참조하세요.
</pre>
- Amazon Keyspaces (for Apache Cassandra)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Keyspaces(Apache Cassandra용)는 확장 가능하고 가용성이 뛰어나며 관리되는 Apache Cassandra 호환 데이터베이스 서비스입니다.
</pre>
- Amazon Neptune
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Neptune은 빠르고 안정적이며 완벽하게 관리되는 그래프 데이터베이스 서비스로, 고도로 연결된 데이터 세트로 작동하는 애플리케이션을 쉽게 구축하고 실행할 수 있습니다. Neptune의 핵심은 수십억 개의 관계를 저장하고 밀리초 단위의 대기 시간으로 그래프를 쿼리하는 데 최적화된 특수 목적의 고성능 그래프 데이터베이스 엔진입니다. Neptune은 널리 사용되는 그래프 쿼리 언어인 Apache TinkerPop Gremlin과 W3C의 SPARQL을 지원하므로 고도로 연결된 데이터 세트를 효율적으로 탐색하는 쿼리를 구축할 수 있습니다. Neptune은 추천 엔진, 사기 탐지, 지식 그래프, 약물 발견, 네트워크 보안과 같은 그래프 사용 사례를 지원합니다.

Amazon Quantum Ledger Database(Amazon QLDB)는 중앙의 신뢰할 수 있는 기관이 소유한 투명하고 불변이며 암호화로 검증 가능한 트랜잭션 로그를 제공하는 완전 관리형 원장 데이터베이스입니다. Amazon QLDB를 사용하면 모든 애플리케이션 데이터 변경 사항을 추적하고 시간 경과에 따른 완전하고 검증 가능한 변경 내역을 유지할 수 있습니다.

AWS WAF는 보호되는 AWS 리소스로 전달되는 웹 요청을 모니터링하고 관리할 수 있는 웹 애플리케이션 방화벽입니다. AWS WAF를 사용하면 Amazon CloudFront 배포, Amazon API Gateway REST API, Application Load Balancer 및 AWS AppSync GraphQL API와 같은 리소스를 보호할 수 있습니다. AWS WAF를 사용하면 웹 요청이 요청이 시작된 IP 주소, 특정 요청 구성 요소의 값, 요청이 전송되는 속도 등 지정한 조건과 일치하는지 검사할 수 있습니다. AWS WAF는 일치 요청을 계산하고, 차단 또는 허용하고, CAPTCHA 퍼즐과 같은 챌린지를 클라이언트 사용자나 브라우저에 보내는 등 다양한 방법으로 일치 요청을 관리할 수 있습니다.
</pre>

### 서버리스
- AWS AppSync
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
AWS AppSync는 실시간 데이터 동기화 및 오프라인 프로그래밍 기능을 갖춘 엔터프라이즈 수준의 완전 관리형 GraphQL 서비스입니다.
</pre>
- AWS Fargate
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Elastic Container Service(Amazon ECS)는 Amazon EC2 인스턴스 클러스터에서 Docker 컨테이너를 쉽게 실행, 중지 및 관리할 수 있게 해주는 확장성이 뛰어나고 빠른 컨테이너 관리 서비스입니다.
</pre>
- AWS Lambda
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
AWS Lambda를 사용하면 서버를 프로비저닝하거나 관리하지 않고도 코드를 실행할 수 있습니다. 사용한 컴퓨팅 시간에 대해서만 비용을 지불하면 되며, 코드가 실행되지 않을 때는 요금이 부과되지 않습니다. 관리할 필요 없이 거의 모든 유형의 애플리케이션 또는 백엔드 서비스에 대한 코드를 실행할 수 있습니다. 코드를 업로드하기만 하면 Lambda가 고가용성으로 코드를 실행하고 확장하는 데 필요한 모든 것을 처리합니다.
</pre>

### 스토리지
- AWS Backup
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
AWS Backup은 클라우드와 온프레미스의 AWS 서비스 전반에 걸쳐 데이터 백업을 쉽게 중앙 집중화하고 자동화할 수 있게 해주는 완전관리형 백업 서비스입니다.
</pre>
- Amazon Elastic Block Store(Amazon EBS)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Elastic Block Store(Amazon EBS)는 Amazon Elastic Compute Cloud 인스턴스에 사용할 블록 수준 스토리지 볼륨을 제공하는 웹 서비스입니다. Amazon EBS 볼륨은 실행 중인 모든 인스턴스에 연결하여 하드 드라이브처럼 사용할 수 있는 가용성이 높고 안정적인 스토리지 볼륨입니다.
</pre>
- Amazon Elastic File System(Amazon EFS)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon EFS는 관리나 프로비저닝이 필요 없이 파일을 추가하고 제거하면 자동으로 커지거나 줄어드는 간단하고 탄력적이며 서버리스이고 탄력적인 설정 후 잊어버리는 파일 시스템입니다. Amazon EC2, AWS Lambda, Amazon ECS, Amazon EKS 및 기타 AWS 컴퓨팅 인스턴스 또는 온프레미스 서버에서 Amazon EFS를 사용할 수 있습니다.
</pre>
- Amazon FSx(모든 유형)
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon FSx를 사용하면 클라우드에서 기능이 풍부한 고성능 파일 시스템을 쉽고 비용 효율적으로 시작, 실행 및 확장할 수 있습니다. 안정성, 보안, 확장성 및 다양한 기능을 통해 광범위한 워크로드를 지원합니다. Amazon FSx를 사용하면 널리 사용되는 4가지 파일 시스템인 Lustre, NetApp ONTAP, OpenZFS 및 Windows File Server 중에서 선택할 수 있습니다. Amazon 파일 캐시는 데이터가 저장된 위치에 관계없이 파일 데이터를 더 쉽게 처리할 수 있게 해주는 AWS의 고속 캐시입니다.
</pre>
- Amazon S3
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
Amazon Simple Storage Service(Amazon S3)는 인터넷용 스토리지입니다. Amazon S3를 사용하면 웹 어디에서나 언제든지 원하는 양의 데이터를 저장하고 검색할 수 있습니다.
</pre>
- Amazon S3 Glacier
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
- Amazon S3 Glacier (S3 Glacier) 는 저렴한 데이터 보관 및 장기 백업을 위한 안전하고 내구성 있는 서비스입니다.
</pre>
- AWS Storage Gateway
<pre style="white-space: pre-wrap; padding: 10px 10px 10px 40px; background-color: #dddddd;">
AWS Storage Gateway는 온프레미스 소프트웨어 어플라이언스를 클라우드 기반 스토리지와 연결하여 온프레미스 IT 환경과 AWS 클라우드의 AWS 스토리지 인프라 간의 원활하고 안전한 통합을 제공하는 서비스입니다.
</pre>
